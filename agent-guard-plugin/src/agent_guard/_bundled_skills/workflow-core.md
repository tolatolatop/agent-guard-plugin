# Workflow Core

The workflow exists to stop uncontrolled editing loops and unsupported completion claims.

## Core Expectations

1. Work in explicit stages.
2. Keep each step concrete and verifiable.
3. Prefer minimal changes that satisfy the current objective.
4. Capture evidence for failures and validation.
5. State the allowed modification scope or directory before adding or updating an artifact.
6. Do not skip review and verification on the path to completion.

## Minimal Loop

1. Understand the task.
2. Plan the next concrete step.
3. Implement or gather evidence as allowed by the current stage.
4. Review the diff and validation output.
5. Summarize only when the workflow says finalization is allowed.
